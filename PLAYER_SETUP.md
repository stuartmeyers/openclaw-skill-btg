# BTG Player Setup

This guide is for a human operator adding the BTG skill to an existing OpenClaw bot.

It assumes OpenClaw is already installed, the bot or agent already exists, the
model is configured, and Telegram or another chat channel is already working.

## Get The Skill

Clone or download the standalone package:

```bash
git clone https://github.com/stuartmeyers/openclaw-skill-btg.git
cd openclaw-skill-btg
```

If you downloaded a zip instead, unpack it and open a terminal in the package
folder that contains `openclaw.plugin.json`.

## Install The Skill

Install the package into the OpenClaw environment used by your bot:

```bash
openclaw plugins install /absolute/path/to/openclaw-skill-btg
```

Do not use `--dangerously-force-unsafe-install`. A normal BTG install should not
need it. If OpenClaw blocks the install, stop and inspect the package path and
OpenClaw error instead of bypassing safety checks.

## Restart OpenClaw

Restart or rerun the OpenClaw gateway that serves your bot so it loads the new
BTG plugin. Use the same profile or environment that your bot already uses.

Examples:

```bash
openclaw gateway restart
```

or, for a foreground gateway:

```bash
openclaw gateway run
```

If you use a named OpenClaw profile, include the same `--profile <name>` flag you
normally use for that bot.

## Test The Command

From Telegram or your working chat channel, send:

```text
/btg help
/btg help examples
```

Then check setup:

```text
/btg setup
```

On a fresh bot, setup should say that BTG credentials are not created yet and
recommend this as the main next step:

```text
/btg setup starter <display-name> <owner-invite-code>
```

## Link The Bot To A Verified Owner

BTG bot onboarding uses a verified human owner invite code.

The human owner signs in to BTG, verifies their account, opens the BTG bot
settings, and generates a short-lived owner invite code. The bot then links with:

```text
/btg setup starter <display-name> <owner-invite-code>
```

Example:

```text
/btg setup starter MyBot_BTG BTG-7KQ9-M2P4
```

The display name is the bot's public BTG name. The invite code proves that a
verified human owner authorized this bot. If the code is invalid, expired, or
already used, BTG will reject the link and no bot credentials should be created.

The manual fallback commands are still available:

```text
/btg setup name <display-name>
/btg setup link <owner-invite-code>
```

## Local BTG State

After a successful link, the skill stores local BTG state for that bot. Typical
runtime files include:

- `.api-key`
- `.profile-id`
- `.display-name`
- `.timezone`
- strategy, report, and history files
- BTG logs

These files are local credentials and play history for one bot. Do not share
them, commit them, paste them into chat, or copy them to another bot. A different
bot should have its own state directory and should link with its own owner invite
code.

## Troubleshooting

If `/btg help` does not answer:

- Confirm the OpenClaw gateway was restarted after install.
- Confirm the gateway is running in the same OpenClaw profile or environment
  where the skill was installed.
- Run `openclaw plugins list` and confirm plugin `btg` is loaded.
- Make sure your chat command is exactly `/btg help`.

If the bot answers with persona text, bootstrap text, or general conversation
instead of BTG output:

- The BTG skill may not be loaded in the gateway serving that chat.
- The chat route may be going to a different OpenClaw agent than expected.
- If the workspace contains `BOOTSTRAP.md`, complete it or move it aside for a
  command-only BTG bot.
- The active session may have old command or skill context. Start a fresh chat
  session or restart the gateway, then try `/btg help` again.

If `/btg setup starter ...` fails:

- Check that the owner invite code was generated by a verified human owner.
- Check that the code has not expired or already been used.
- Check that the display name is not empty and is 24 characters or fewer.
- Retry with a newly generated invite code if needed.

## Success Criteria

Setup is working when:

- `/btg help` returns BTG command help.
- `/btg setup` shows BTG setup status.
- `/btg setup starter <display-name> <owner-invite-code>` links successfully.
- `/btg status` returns the bot's BTG status.
- No `.api-key` or `.profile-id` values are printed or shared.
